## Online Matching Framework

`matching_framework.py` provides a unified `EventDrivenMatchingSystem` class supporting
four strategies selectable via the `model_type` parameter:

| `model_type` | Strategy |
|--------------|----------|
| `'Proposed'` | Cloud-model parameters + dynamic reservation threshold + information unreliability penalty + 3En fuzzy early exit |
| `'FCFS'` | First-Come, First-Served |
| `'Greedy-Det'` | Deterministic greedy (expectation values only) |
| `'Greedy-TFN'` | Greedy matching with triangular fuzzy number defuzzification |

### Quick start

```python
from matching_framework import EventDrivenMatchingSystem

# Run the proposed framework
sys = EventDrivenMatchingSystem('demand.csv', 'supply.csv', model_type='Proposed')
sys.run_simulation()
metrics = sys.evaluate_metrics()
print(metrics)
# {'CR': 0.5089, 'Eff_RSR': 0.9083, 'AWT (h)': 4.47, 'Eff_Gini': 0.0917, 'NIR': 0.92, 'Model': 'Proposed'}
```

### Evaluation metrics

| Metric | Description |
|--------|-------------|
| `CR` | Competitive ratio — ratio of online cumulative utility to offline optimal |
| `Eff_RSR` | Effective resource satisfaction rate among genuinely valid demanders |
| `AWT (h)` | Average waiting time from demander posting to match establishment |
| `Eff_Gini` | Effective fairness (Gini coefficient) among genuinely valid demanders |
| `NIR` | Noise immunity rate — proportion of semantic-conflict demanders correctly isolated |

## Dataset Generation

`generate_datasets.py` produces four families of synthetic datasets from the baseline
of 159 demanders and 68 suppliers:

| Family | Datasets | Target dimension |
|--------|----------|-----------------|
| Scale Expansion | Expanded_10×, 100×, 1000× | Bootstrap resampling with small Gaussian noise |
| Resource Scarcity | Scarcity_75, 50, 25, 10 | Supply quantity scaled by λ ∈ {0.75, 0.50, 0.25, 0.10} |
| Demand Backlog | Backlog_25, 50, 75 | Arrival times redirected to a delayed release window |
| Semantic Conflict | Conflict_15, 30, 50 | Text-level injection (Type A: exaggeration, Type B: contradiction, Type C: false information) |

Each generation rule perturbs only its target dimension; all other parameters are
strictly preserved. Statistical fidelity is verified by two-sample KS tests in the
paper (Appendix B). The Conflict datasets modify only the unstructured text field;
cloud-model parameters must be re-extracted by feeding the modified texts through
the LLM virtual sampling pipeline described in the paper.

```bash
python generate_datasets.py
```
